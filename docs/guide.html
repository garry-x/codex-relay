codex-relay

OpenAI Codex CLI 代理包装器

Node.js ≥ 18 零依赖

概述

codex-relay 提供两种代理模式，解决特定网络环境下 codex 无法直接访问 OpenAI API 的问题。

直连模式

codex ── 静态代理 ── OpenAI

本机直接访问静态 HTTP 代理，通过它连接 OpenAI。配置后 run 命令自动注入环境变量。

链式中转模式

codex ── 中转服务器 ── 静态代理 ── OpenAI

本机无法直连静态代理时，通过中间服务器转发。中转服务器做 token 认证和代理转发。

安装

# 下载脚本后执行安装（自动选择可写 bin 目录）
./codex-relay install

# 强制覆盖安装
./codex-relay install --force

# 仅更新 Codex CLI
./codex-relay install --update

依赖：Node.js ≥ 18、curl、npm。安装后确认目标目录在 PATH 中。

模式一：直连

配置流程

  1. 配置上游代理 
    codex-relay proxy set http://user:pass@your-proxy.com:8080
    配置持久化到 ~/.codex-relay/config.json。
  2. 验证连通性 
    codex-relay proxy check
    输出示例： 
    Proxy:    http://user:pass@host:47453
Test URL: https://ipv4.icanhazip.com
Status:   200
Latency:  1153ms
Result:   OK — proxy is reachable
  3. 运行 codex 
    codex-relay run
# 或直接透传
codex-relay chat

涉及命令

命令用途
proxy set <url>配置上游代理，持久化保存
proxy show查看当前配置和生效的代理
proxy unset清除代理配置
proxy check [--url URL] [--timeout S]全面诊断：代理配置、连通性（延迟+状态码）、npm、codex
run <args...>注入代理环境变量并运行 codex

模式二：链式中转

中间服务器 部署中转服务

  1. 安装 codex-relay 
    ./codex-relay install
  2. 生成访问 token 
    codex-relay chain token generate
AbCdEf1234567890...  ← 复制此 token，仅输出一次
    Token 以 SHA-256 hash 形式保存在配置文件中，不会明文存储。
  3. 保存中转配置 
    # 基础配置：多个上游按顺序 fallback，direct 表示直连目标
codex-relay chain config \
  --listen 0.0.0.0:8080 \
  --upstream http://user:pass@static-proxy:8080,direct

# 可选：启用 TLS
codex-relay chain config \
  --listen 0.0.0.0:8443 \
  --upstream http://user:pass@static-proxy:8080,direct \
  --tls-cert /etc/codex-relay/fullchain.pem \
  --tls-key /etc/codex-relay/privkey.pem
  4. 启动中转服务 
    codex-relay chain start
codex-relay chain status
chain relay:    running (pid 12345)

本机 配置客户端

  1. 安装 codex-relay（如未安装） 
    ./codex-relay install
  2. 配置代理指向中间服务器 
    # token 放在 URL 的 user 部分，以 Proxy-Authorization 传递
codex-relay proxy set http://<TOKEN>@relay.example.com:8080

# 如果中间服务器启用了 TLS
codex-relay proxy set https://<TOKEN>@relay.example.com:8443
  3. 验证连通性 
    codex-relay proxy check
  4. 运行 codex 
    codex-relay run

中转服务器管理命令

命令用途
chain config --listen --upstream保存中转配置
chain token generate生成 token，hash 写入配置
chain token show / unset查看 / 清除 token
chain start / stop / restart管理后台守护进程
chain status查看守护进程状态和心跳
chain logs查看请求日志
chain serve --listen --upstream [--token]前台运行（调试用）

请求链路

直连模式

# codex-relay run 设置环境变量 HTTP_PROXY=http://proxy:8080 # codex 发出请求 codex ──CONNECT api.openai.com:443── 静态代理 ──TLS 隧道── OpenAI

链式中转模式

# codex-relay run 设置环境变量 HTTP_PROXY=http://token@relay:8080 # 请求流程 codex ──CONNECT api.openai.com:443── chain relay Proxy-Authorization: Basic <token> chain relay ──CONNECT api.openai.com:443── 静态代理 Proxy-Authorization: Basic <upstream-auth> 静态代理 ──TLS 隧道── OpenAI

链式中转服务器不修改请求目标、不做 DNS 解析、不做 TLS 解密。仅做 token 校验和认证 header 替换。

环境变量

run 命令自动注入以下环境变量到 codex 进程：

变量值
HTTP_PROXY配置的代理 URL
HTTPS_PROXY配置的代理 URL
WS_PROXY配置的代理 URL
WSS_PROXY配置的代理 URL
NO_PROXYlocalhost,127.0.0.1,::1,.local

代理优先级：HTTP_PROXY 环境变量 > proxy set 配置。
